🔄 以旧换新 — 转化率分析
方案流程路径转化障碍转化评估
ETC助手 首页 → 旧设备估价 → 选择设备 → 支付页 → 支付成功 → 后续履约 支付成功页多一跳出点；估价与设备选择分两步，流程略长 中
速通 首页 → 旧设备估价 → 选择设备 → 支付页 → 支付成功 → 后续履约 支付成功页多一跳出点；估价结果页展示不够直观，抵扣感知弱 中
本方案 设备升级首页 → 旧设备估价 → 选择新设备 → 确认地址 → 直接支付 已移除：支付成功页、确认订单页、手动添加车牌 —
📌 本方案设计思路
核心逻辑：「先展示价值，再引导付款」。用户在评估结果页看到原价、抵扣金额、实付价后，购买意愿最强，此时引导地址填写+直接支付转化率最高。
关键设计决策：
· 自动读取已绑定车牌，消除手动输入摩擦
· 仅支持有旧设备场景——无旧设备用户抵扣为 ¥0，展示换新意义不大，引流至新购避免流失
· 移除支付成功中间页，支付后直达订单详情，减少跳出点
🛒 新购 — 转化率分析
方案流程路径转化障碍转化评估
乐速通
前置审核		 选设备 → 填车牌 → 提交行驶证等资料 → 审核通过后才发货 支付前必须证件审核，等待期用户大量流失；审核失败直接截断转化 低
ETC助手
无任何验证		 选设备 → 填车牌（无验证）→ 填地址 → 支付 → 后续履约 车牌无验证存在合规风险；支付成功页多一跳出点 高
支付宝
平台流量入口		 搜索/推荐 → 选设备 → 填车牌+地址 → 支付宝支付 → 后续履约 依赖支付宝平台流量，品牌感弱；车牌填写步骤增加摩擦；跨平台跳转易流失 中
货车ETC
货车专属场景		 货车入口 → 填车牌（验证货车资质）→ 选设备 → 提交资料 → 支付 → 后续履约 目标用户为货车司机，资质验证更严格；前置车牌+资质核验，转化漏斗损耗大 低
山东高速
省级闭环		 官方渠道入口 → 填车牌（实名验证）→ 选设备 → 填地址 → 支付 → 发货 实名验证门槛高；地域限制强（仅山东发行用户）；品牌可信度高但覆盖面窄，整体转化规模有限 中
本方案 办理首页 → 选设备 → 确认地址（无需车牌）→ 直接支付 → 订单详情（支付后再完成资料/签约/激活） 已移除：车牌填写、支付成功页；资料后置，支付前零门槛 —
📌 本方案设计思路
核心逻辑：「支付前零门槛，支付后强任务」。将资料审核、车辆签约、激活全部后置到订单详情任务清单，支付前只需确认地址，最大化漏斗顶部转化率。
关键设计决策：
· 无需在支付前填写车牌，彻底消除资质验证摩擦
· 移除支付成功页，支付后直达待收货订单详情，减少跳出
· 资料审核后置并后置签约：资料审核通过即可签约（无需等收货），激活才需要收到设备——通过任务清单驱动用户主动完成，不影响支付转化
· 风险对冲：订单强任务清单保障后续激活完成率，弥补前置无验证的合规缺口
